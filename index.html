Project Summary

สรุปแนวคิดและขอบเขตโครงการ (สรุปตามรายงานบทที่ 1 และบทที่ 3)

โครงการนี้มุ่งพัฒนา “เว็บแอปพลิเคชันทำนายราคาทองคำรายวันแบบเรียลไทม์” เพื่อช่วยให้ผู้ใช้งานเข้าถึงผลการพยากรณ์ที่อัปเดตต่อเนื่อง โดยกำหนดเป้าหมายการพยากรณ์เป็นราคาทองคำของ “วันถัดไป (t+1)” ครอบคลุม Gold High, Gold Low และ Gold Close (USD/oz) ภายใต้แนวคิดการนำข้อมูลการเงินรายวันและปัจจัยมหภาคมารวมกันเพื่อเพิ่มความแม่นยำในการทำนาย

ข้อมูลที่ใช้ประกอบด้วยข้อมูลรายวันจาก Yahoo Finance (เช่น ราคาทองคำ, ปริมาณการซื้อขาย, สินทรัพย์/ดัชนีที่เกี่ยวข้อง) และข้อมูลเศรษฐกิจมหภาคจาก FRED (เช่น CPI) ซึ่งมีความถี่รายเดือนจะถูกปรับให้อยู่ในกรอบรายวันด้วยวิธี forward fill (ffill) จากนั้นทำการรวมข้อมูลตามแกนเวลา ทำความสะอาดข้อมูล และสร้างคุณลักษณะแบบ lag เพื่อทำนายแบบ one-step-ahead

ในด้านตัวแบบ โครงการเปรียบเทียบหลายแนวทาง ได้แก่ Linear Regression, CatBoost, XGBoost, LightGBM และ LSTM ภายใต้กรอบประเมินแบบอนุกรมเวลา (walk-forward) ร่วมกับการปรับแต่งไฮเปอร์พารามิเตอร์ด้วย TimeSeriesSplit และ Grid Search โดยใช้เมตริกมาตรฐาน เช่น RMSE, MAE, MAPE และ R² เพื่อคัดเลือกโมเดลที่เหมาะสมที่สุดสำหรับนำไปใช้งานจริง

เพื่อให้ผลการพยากรณ์โปร่งใสและตีความได้ โครงการประยุกต์ใช้ SHAP เพื่ออธิบายความสำคัญของตัวแปรทั้งระดับภาพรวม (global) และระดับรายกรณี (local) และนำผลลัพธ์ทั้งหมดไปแสดงบนเว็บแอปแบบ Client–Server โดยใช้ FastAPI เป็น backend และ React + Chart.js เป็น frontend พร้อมตั้งค่าให้อัปเดตข้อมูลอัตโนมัติผ่าน GitHub Actions และ deploy ระบบบน Render

Workflow

End-to-end pipeline: data → features → models → SHAP → web app (real-time)

1

Automated Data Ingestion

ดึงข้อมูลจาก Yahoo Finance (ผ่าน yfinance) และ FRED (ผ่าน fredapi) และตั้ง schedule ให้อัปเดตอัตโนมัติ (เช่น รายวัน) ด้วย GitHub Actions

2

Data Merging & Cleaning

รวมข้อมูลหลายแหล่งตาม Date, เติมค่าข้อมูลต่างความถี่ด้วย ffill, จัดรูปแบบวันที่, จัดการ missing และ export เป็น CSV สำหรับฝึก/ใช้งานระบบ

3

Feature Engineering (Lag / Scaling)

สร้างคุณลักษณะ lag (เช่น lag 1), คัดเลือกตัวแปรตามเกณฑ์ (เช่น correlation/VIF/RFE สำหรับเชิงเส้น), เตรียมรูปแบบข้อมูลสำหรับ LSTM (windowing)

4

Model Training & Walk-forward Evaluation

แบ่ง Train/Val/Test ตามลำดับเวลา, จูนพารามิเตอร์ด้วย TimeSeriesSplit + Grid Search และประเมินแบบ walk-forward (one-step-ahead) ด้วย RMSE/MAE/MAPE/R²

5

Explainability with SHAP

สร้างคำอธิบายตัวแบบทั้ง global (feature importance) และ local (อธิบายแต่ละวัน/แต่ละครั้งที่ทำนาย) เพื่อเพิ่มความโปร่งใสในการใช้งานจริง

6

Real-time Serving & Dashboard

Backend ด้วย FastAPI โหลดโมเดลและให้บริการ REST API; Frontend ด้วย React + Chart.js เรียกผลทำนายและแสดงกราฟ/ตาราง รวมถึงส่วน SHAP dashboard และ deploy บน Render

Systems Design

Client–Server architecture + automation (data update) + model transparency (SHAP)

Architecture

  • Targets: Gold High / Gold Low / Gold Close (t+1)
  • Data Sources: Yahoo Finance (daily) + FRED (macro; monthly → daily via ffill)
  • Preprocessing: merge by Date, handle missing, standardize/scale (as needed)
  • Models: Linear Regression, CatBoost, XGBoost, LightGBM, LSTM
  • Evaluation: Time-based split + walk-forward, metrics: RMSE/MAE/MAPE/R²
  • XAI: SHAP global + local explanations
  • Backend: FastAPI (predict endpoints, optional /docs)
  • Frontend: React (SPA) + Chart.js for visualization
  • Automation: GitHub Actions schedule for daily data update
  • Deploy: Render (Backend service + Frontend static site)

Deliverables

Daily Forecast (t+1) Walk-forward Evaluation SHAP Dashboard FastAPI REST API React + Chart.js UI GitHub Actions ETL Render Deployment
